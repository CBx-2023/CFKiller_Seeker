# SPOT.ph Eat + Drink 抓取脚本使用说明

## 概述

自动抓取 SPOT.ph 网站 Eat + Drink 频道的所有文章，包括标题、日期、作者、正文、图片链接等。

<span style="color:#FF6666;">注意：当前没有写截图功能，截图功能要自己写</span>

脚本采用 IndexedDB 中转方案：边滚动边抓取，每篇文章立即写入浏览器 IndexedDB（磁盘），不在 JS 内存中堆积。全部完成后一次性导出为 JSON 文件下载。

## 文件位置

```
spotph_scraper_v6.js        # 完整版（带注释）
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| index | number | 文章序号 |
| title | string | 文章标题 |
| url | string | 文章链接 |
| date | string | 发布日期 |
| author | string | 作者 |
| scraped_at | string | 采集时间（ISO 8601） |
| images | array | 图片 URL 列表 |
| imageCount | number | 图片数量 |
| content | string | 正文全文（段落用 \n\n 分隔） |
| paragraphs | number | 段落数量 |
| error | string | 仅失败时出现，错误信息 |

## 工作原理

```
┌─────────────────────────────────────────────────┐
│  浏览器页面                                      │
│                                                  │
│  1. 自动向下滚动 → 触发懒加载更多文章              │
│  2. 提取新出现的文章链接                           │
│  3. 逐篇 fetch 获取 HTML → 解析 → 提取字段         │
│  4. 每篇立即写入 IndexedDB（磁盘）→ 释放内存        │
│  5. 重复直到没有新文章加载                         │
│  6. 全部完成后从 IndexedDB 导出 → 排序 → 下载 JSON  │
│  7. 清理 IndexedDB                                │
└─────────────────────────────────────────────────┘
```

## 内存占用

| 阶段 | JS 内存占用 |
|------|------------|
| 抓取中 | 极低（始终只有 1 篇在内存） |
| 导出时 | 约等于文件大小（21篇 ≈ 200KB，10000篇 ≈ 50MB） |
| 完成后 | 归零（IndexedDB 已清理） |

## 预估时间与大小

| 文章数 | 预估耗时 | 文件大小 |
|--------|---------|---------|
| 21 篇 | ~30 秒 | ~200 KB |
| 100 篇 | ~2 分钟 | ~1 MB |
| 500 篇 | ~10 分钟 | ~5 MB |
| 1000 篇 | ~20 分钟 | ~10 MB |
| 10000 篇 | ~3 小时 | ~50-80 MB |

## 注意事项

1. <span style="color:#FF6666;">当前没有截图功能，截图功能要自己写</span>
1. **网络要求**：需要能正常访问 spot.ph
2. **浏览器要求**：Chrome / Edge（需要支持 IndexedDB）
3. **不要关闭页面**：脚本运行期间不要关闭或刷新页面
4. **反爬风险**：每篇间隔 600ms，大量抓取时可能触发网站限流
5. **断点续抓**：当前版本不支持中断恢复，如需可手动改造

## 故障排除

**Q: 控制台报错 `Unexpected end of input`**
A: 脚本没有完整粘贴，请确保从 `(async` 到最后的 `})()` 全部复制。

**Q: 抓取过程中页面卡住**
A: 正常现象，脚本在等待滚动加载。如需中止，刷新页面即可。

**Q: 下载的 JSON 文件为空**
A: 可能是页面未完全加载就执行了脚本，刷新页面后重试。

**Q: 大量文章抓取被网站拦截**
A: 增大脚本中的 `delay(600)` 值，改为 `delay(2000)` 或更大。
